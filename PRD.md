# AI 导航站 - 产品需求文档 (PRD)

**版本**: v3.0
**更新日期**: 2026-03-01
**状态**: 规划中

---

## 一、产品定位

### 1.1 核心定位

> **AI 导航站 — 想了解 AI、进入 AI 生态，第一个想到的网站**

目标用户是 **AI 入门的普通人**——听说过 AI 但不知道从哪里开始，想用 AI 解决实际问题但面对繁杂工具无从下手。

不面向开发者、研究者等深度用户（他们有自己的社区）。面向的是真正的普罗大众，这是当前 AI 时代最大的未被服务群体。

### 1.2 平台角色

平台不是内容创作者，也不是内容搬运者，而是：

```
策展人 + 导航员

发现优质内容/工具 → 筛选过滤 → 打标签组织 → 呈现给用户 → 跳转至原始来源
```

用户在平台上找到"入口"，实际内容/工具在原始网站消费。平台的价值在于**降低发现成本**和**降低使用门槛**。

### 1.3 战略方向

| 阶段 | 策略 | 核心目标 |
|------|------|---------|
| 当前 | **先做宽** | 覆盖 AI 入门用户所有需求，积累流量和用户习惯 |
| 中期 | **做深护城河** | 社区数据、独家关系、用户身份资产不可迁移 |
| 长期 | **做厚变现** | 广告收入 + 自有/联合 AI 产品发布 |

### 1.4 变现路径

**路径一：广告营销收入**
- AI 工具赞助位（精准用户，单次价值高于泛流量）
- 新产品首发推广（独家报道 + 导流）
- Newsletter/邮件赞助

**路径二：产品发布与联合发布**
- 基于平台数据洞察，发布自有 AI 产品
- 成为中国 AI 新品"首发平台"，与项目方共享用户和分成
- 海外 AI 工具本地化分发代理

## 二、五大核心模块

### 2.1 邀请码接龙（引流钩子）

**定位**：平台差异化引流功能，利用稀缺性驱动注册和分享。

**核心逻辑**：
- 邀请码是真实稀缺资源，积分消耗机制有合理性
- 查看邀请码：-50 积分（唯一的积分消耗点）
- 提供邀请码被查看：+30 积分（供给侧激励）
- 新用户注册即获 100 积分，天然覆盖 2 次邀请码查看

**功能边界**：
- 邀请码与 AI 工具绑定，在工具详情页内呈现
- 接龙列表展示（模糊预览），登录后可花积分查看明文
- 接龙只有一层（防止过度复杂）
- 保留现有技术实现，不作大改

**注意**：邀请码需求随 AI 产品开放注册而自然萎缩。此模块作为**引流和冷启动工具**，不是平台长期核心。

### 2.2 AI 工具库（导航核心）

**定位**：帮用户找到最适合自己需求的 AI 工具，是平台最基础的价值。

**内容来源**：
- 参考 watcha.cn 等工具聚合站，深度过滤
- 官方认领机制：AI 工具方可认领并维护自己的页面
- 用户提交新工具（管理员审核后上线）

**过滤标准**（只收录通过全部条件的工具）：
```
必须通过：
  ✓ 中国用户可访问（或有明确可访问方案）
  ✓ 已正式发布，非内测/实验版
  ✓ 有免费版或低门槛试用入口
  ✓ 产品稳定，非临时性项目

直接排除：
  ✗ 纯英文且无中文支持
  ✗ 需要海外信用卡订阅且无替代方案
  ✗ 仅适合开发者，无普通用户使用场景
```

**工具卡片信息**：

| 字段 | 说明 |
|------|------|
| 名称、Logo、官网链接 | 基础信息 |
| 一句话功能描述 | 面向普通用户语言，非功能列表 |
| 适合谁用 | 如："适合写作者，不需要编程" |
| 能做什么 | 3 个具体使用场景 |
| 访问方式 | 直接访问 / 需要工具 / 需要邀请码 |
| 免费额度 | 明确说明免费能用多少 |
| 定价 | 付费方案价格（人民币/美元） |
| 分类标签 | 对话 / 绘图 / 编程 / 视频 / 写作 / 搜索 / 效率 / Agent |
| 是否有邀请码 | 链接至邀请码接龙 |
| 收藏数 | 用户收藏量 |

**工具分类**：
```
对话 AI | 绘图 AI | 视频 AI | 写作 AI | 代码 AI
搜索 AI | 效率工具 | Agent 工具 | 音乐 AI | 其他
```

### 2.3 知识库（策展导航）

**定位**：AI 领域优质内容的发现与导航，帮用户找到有价值的深度文章、教程和资料。

**核心原则**：
- 平台**不创作内容**，只负责发现、筛选、打标签、呈现
- 用户点击详情直接跳转原文链接，不在平台内展示全文
- 偏好**深度、长期有效的内容**（入门教程、场景指南），而非实时新闻

**内容来源**：

| 来源 | 方式 | 审核 |
|------|------|------|
| 编辑主动收录 | Twitter 优质长帖、Medium、少数派等人工发现 | 直接上线 |
| 用户投稿 | 用户提交链接 + 简介 | 后台审核后上线 |
| 自动抓取工具 | RSS / 指定源自动抓取候选 | 后台人工筛选确认上线 |

**内容分类**：
```
大模型      → GPT/Claude/Gemini 等使用教程、对比、评测
Agent框架   → OpenClaw/AutoGen/CrewAI 等框架入门和实践
MCP         → Model Context Protocol 相关内容
教程        → 零基础入门、场景化使用指南
工具评测    → AI 工具深度评测和横向对比
行业洞察    → AI 趋势分析、商业模式解读（非实时新闻）
```

**内容卡片字段**：
```
标题（可由编辑优化，更面向普通用户）
2句话摘要（讲什么 + 适合谁）
来源标注（Twitter/@xxx / Medium / 少数派 / ...）
分类 + 难度（入门 / 进阶 / 专业）
标签
发布/收录时间
[去看原文 →]（主要操作，直接跳转）
[收藏]
```

**积分激励**：
- 用户投稿被收录：+20 积分
- 投稿内容被收藏 10 次：+15 积分额外奖励

### 2.4 提示词广场（UGC 社区）

**定位**：社区共创的提示词工程库，帮普通用户学会"如何让 AI 做事"。

> 注：原名"技能广场"，重命名为"提示词广场"，更准确反映内容性质。

**内容特征**：
- 纯 UGC，用户提交，社区点赞/收藏驱动内容质量筛选
- 强调实用性：可以直接复制使用的提示词，不是理论
- 有场景分类，方便按需查找

**提示词卡片字段**：
```
标题（一句话说明用途）
分类（写作/编程/设计/营销/分析/翻译/教育/职场/生活）
适用产品（ChatGPT/Claude/Gemini 等）
提示词预览（前 2 行，详情页展开全文）
标签
点赞数、收藏数、使用次数
提交者昵称
[复制提示词] [收藏] [查看详情]
```

**积分激励**：

| 行为 | 积分 |
|------|------|
| 提交提示词（审核通过） | +15 |
| 提示词被收藏 10 次 | +20 |
| 提示词被点赞 50 次 | +30 |

### 2.5 Agent 技能库（MCP 目录）

**定位**：Agent Skills 和 MCP Server 的中文目录，是目前市面上稀缺的内容资产。

**背景**：MCP（Model Context Protocol）生态正在爆发，但中文领域几乎没有系统化的目录和入门内容。这是一个明确的内容空白，也是高价值 SEO 机会。

**内容类型**：

| 类型 | 说明 | 示例 |
|------|------|------|
| MCP Server | 连接 AI 与外部系统的协议实现 | GitHub MCP、文件系统 MCP、数据库 MCP |
| Agent Skills | 主流 Agent 框架的技能/插件 | OpenClaw Skills、AutoGen Tools |

**面向用户的描述方式**：
- 不是技术文档："GitHub MCP integration for LLM"
- 而是用户价值："让 AI 直接帮你管理 GitHub 仓库"

**技能卡片字段**：
```
名称
类型（MCP Server / Agent Skill）
功能描述（面向普通用户语言）
用一句话能做什么（如"让AI读取你的Google日历"）
适用框架（OpenClaw / AutoGen / 通用 / ...）
难度（入门 / 进阶 / 专业）
安装方式摘要
来源链接（GitHub / 文档）
[查看详情 →] [收藏]
```

**内容来源**：
- 编辑主动收录 GitHub 上的 MCP 项目
- OpenClaw、AutoGen 等框架的官方 Skills 目录
- 社区用户提交

## 三、积分体系

### 3.1 设计原则

- **只有正向积累，唯一消耗点是稀缺资源（邀请码）**
- 积分是"参与凭证"，记录用户在平台的贡献历史
- 早期用户身份具有不可复制性（创始成员身份永久标记）
- 未来价值保持想象空间（会员体系、空投资格等）

### 3.2 早期用户身份

```
第 1 - 1,000 名注册用户     → 「创世成员」永久标识
第 1,001 - 10,000 名       → 「先行者」
第 10,001 - 100,000 名     → 「早期成员」
之后                        → 普通会员
```

这些身份无法通过后续行为获得，只能靠"来得早"。

### 3.3 积分获取（全量）

**注册与基础**

| 来源 | 积分 | 频率 |
|------|------|------|
| 注册赠送 | +100 | 一次性 |
| 每日签到 | +5 | 每日 |

**一次性新手任务**

| 任务 | 积分 |
|------|------|
| 完善昵称/头像 | +10 |
| 发表首条评论 | +15 |
| 首次分享页面至社交媒体 | +20 |
| 首次提交提示词 | +20 |
| 邀请第一位好友注册 | +50 |

**日常任务**

| 任务 | 积分 | 频率 |
|------|------|------|
| 点赞 3 篇内容 | +3 | 每日 |
| 发表 1 条评论 | +8 | 每日 |

**内容贡献**

| 来源 | 积分 |
|------|------|
| 邀请码被查看 | +30 |
| 知识库投稿（审核通过） | +20 |
| 知识库投稿被收藏 10 次 | +15 |
| 提示词提交（审核通过） | +15 |
| 提示词被收藏 10 次 | +20 |
| 提示词被点赞 50 次 | +30 |

**里程碑任务**

| 任务 | 积分 |
|------|------|
| 累计评论 10 条 | +30 |
| 邀请 5 位好友 | +100 |
| 连续签到 7 天 | +30 |

### 3.4 积分消耗

| 用途 | 积分 | 说明 |
|------|------|------|
| 查看邀请码 | -50 | 唯一消耗点，因邀请码是真实稀缺资源 |

> 注册 100 积分 = 2 次邀请码查看，用户开箱即可体验核心价值。

## 四、Onboarding 设计

### 触发条件
- 用户首次注册登录后自动触发
- localStorage 记录状态，确认后永不再出现
- 任意步骤均可点击"跳过"退出

### 步骤设计（最多 4 步）

```
Step 1：这里是什么？
  "AI 导航站 — 帮你找到最适合自己的 AI 工具，从零开始"
  （一句话，配插图）

Step 2：这里有什么？
  三个模块卡片：AI工具库 / 知识库 / 提示词广场
  每个配 1 行说明

Step 3：参与就有积分
  "你的每一次浏览、分享、投稿都会积累积分"
  "积分是你在这里的贡献凭证"
  （不解释积分的具体用途，保留想象空间）

Step 4：开始探索
  → 「去看看 AI 工具」（主按钮）
  → 「先了解一下」（次按钮，跳到知识库）
```

### 设计原则
- Onboarding 结束后，价值传递完全依赖页面布局和 UI 设计
- 如果一个功能需要反复解释，优先改设计，而不是加说明

## 五、首页设计方向

### 首屏核心任务
在 5 秒内回答用户三个问题：
1. 这是什么地方？
2. 对我有什么用？
3. 我第一步做什么？

### 信息架构（从上到下）

```
[首屏 Hero]
  标语：帮你找到最适合自己的 AI 工具
  副标语：工具导航 · 入门教程 · 提示词

[热门 AI 工具]（3-4 个精选卡片）
  → 查看全部工具

[最新知识库文章]（3 篇，入门友好优先）
  → 去知识库

[热门提示词]（3 个，可直接复制）
  → 去提示词广场

[邀请码入口]（轻量展示，不作为首屏核心）
```

### 模块标题规范
```
不写：          写：
热门产品    →   "这些 AI 工具值得你试试"
最新资讯    →   "AI 世界今天发生了什么"
技能广场    →   "一句话让 AI 帮你做任何事"
```

## 六、后台管理系统

**这是 v3.0 阶段最重要的基础设施缺口。**

没有后台管理系统，知识库和 Agent 技能库无法运转。

### 需要支持的管理功能

| 模块 | 功能 |
|------|------|
| 知识库管理 | 文章列表、审核投稿、手动录入、编辑/下架 |
| 工具库管理 | 工具列表、审核用户提交、编辑工具信息 |
| 提示词管理 | 提示词列表、审核投稿、编辑/下架 |
| Agent技能库管理 | MCP/技能录入、编辑、上下架 |
| 用户管理 | 用户列表、积分调整、封禁 |
| 内容抓取管理 | 查看自动抓取候选队列、一键确认上线/拒绝 |

### 访问控制
- 独立后台路由（如 `/admin`），非公开
- 管理员账号单独维护，不与普通用户共用表
- 简单权限：超级管理员 / 内容编辑员

## 九、护城河建立路径

飞轮模型——各环节相互强化，整体难以复制：

```
更多用户 → 更多UGC内容质量 → 更强心智品牌
    ↑                                    ↓
更多AI项目方主动合作 ← 更精准的用户流量价值
```

**四层护城河**：

| 层级 | 内容 | 特点 |
|------|------|------|
| 第一层 | 策展内容资产 | 最基础，可被复制，靠持续积累建立优势 |
| 第二层 | 社区身份与贡献 | 用户积分/声誉/作品不可迁移，转换成本 |
| 第三层 | 用户行为数据 | 真实使用数据对 AI 公司有价值，竞争者无法追赶 |
| 第四层 | 独家供给关系 | 与 AI 项目方建立"首发惯例"，来者无法获得 |
