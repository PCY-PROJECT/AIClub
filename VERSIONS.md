# AI 导航站 - 版本规划

**更新日期**: 2026-02-28

---

## 版本发布原则

- **MVP 优先**：只上线核心链路，不追求完美，快速验证
- **按反馈迭代**：每个版本上线后，根据用户行为数据调整下版优先级
- **数据库向前兼容**：新字段加 DEFAULT，不删改已有字段，降低发布风险

---

## v1.0 MVP — 核心链路上线

**目标**：跑通"注册 → 获得积分 → 浏览产品 → 提交/查看邀请码"完整闭环

### 功能清单

| 模块 | 功能 | 优先级 |
|------|------|--------|

| 用户 | 邮箱登录 / 登出 | P0 |
| 用户 | 防羊毛：IP 限流、邮箱黑名单、新账号冷静期24h | P0 |
| 积分 | 注册赠送100积分 | P0 |
| 积分 | 查看邀请码消耗50积分 | P0 |
| 积分 | 邀请码被查看奖励30积分 | P0 |
| 积分 | 积分流水记录与查看 | P1 |
| 积分 | 每日签到+10积分 | P1 |
| 产品 | 产品列表（分类筛选） | P0 |
| 产品 | 产品详情页 | P0 |
| 邀请码 | 提交邀请码到产品接龙列表 | P0 |

| 邀请码 | 花积分查看邀请码（事务安全） | P0 |

- 用户申请添加产品

## v3.0 — 产品战略升级

**更新日期**: 2026-03-01

**目标**：从 demo 转为正式可运营产品，定位为"AI 入门用户的第一站"

**核心变化**：
1. 资讯模块 → 知识库（策展导航，不创作内容，跳转原文）
2. 技能广场 → 提示词广场（重命名，性质不变）
3. 新增 Agent 技能库（MCP & Agent Skills 目录）
4. 新增后台管理系统（内容审核/上架核心能力）
5. 积分体系重设计（只有正向激励，唯一消耗点是邀请码）
6. 早期用户身份体系（user_no，不可复制）
7. Onboarding 流程
